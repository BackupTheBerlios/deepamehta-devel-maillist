Summary: HSQL-based CM gets corrupted by single quotes (')

Details: When using HSQL-DB as DeepaMehta storage (as in a default installation) the database contents gets corrupted once a single quote character (') is entered in any text field (resp. is acquired programatically, e.g. when a webpage-topic stores HTML content).

This is a serious bug: existing contents are overridden and thus lost. DeepaMehta can't even run anymore.

Eschnepel and jri will meet on Monday, April 21 to fix this problem by adapting the storage layer to SQL prepared statements.
