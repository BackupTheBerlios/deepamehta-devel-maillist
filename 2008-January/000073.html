For the moment it is not easy to delete the open topicmap directly, 
that is the topicmap the user is currently looking at. To delete the 
current topicmap the user must switch to his/her personal workspace 
and delete the closed incarnation of the respective topicmap there. 
But is not always easy to identify the closed topicmap in the personal 
workspace, in particular if there are a lot of topicmap working copies 
obtained by opening a topicmap from a shred workspace many times. 
Thus, the user's personal workspace tends to be cluttered with 
unneeded topicmap working copies.

To alleviate this problem it would be beneficial if an open topicmap 
would provide a "Delete" command in its context menu.
