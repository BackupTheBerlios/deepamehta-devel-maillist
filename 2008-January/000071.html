Summary: Retyping of TopicType produce Server Error

Details: It is only foreseen to retype concrete Instances and not TopicTypes.
If this is done and the TopicMap which includes the former TopicType should be displayed, the Client freeze. When the client Startsup, the Sever logs following Error Message: "*** CorporateDirectives.updateCorporateMemory(): de.deepamehta.DeepaMehtaException: error while accessing the type topic of '....' "
At the moment the only fix if this is already happened is to rewrite the Type of the Topic in the Corporate Memory manually to 'tt-topictype'.
