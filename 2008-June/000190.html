For quality assurance we need an infrastructure for
continous builds:
1) nightly builds from the svn-repository
2) running the testcases (if any)
3) installation tests on various platforms and database
systems
4) performance tests
5) framework tests (servlet and gui)
In a first step a local infrastructure needs to be
created to build and test all that is necessary. As
tools there will be used Maven 2 and VirtualBox.
