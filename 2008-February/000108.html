Summary: Install script forgets to create "hsqldb-intern" directory

Details: The default installation procedure emits an error and quits. DeepaMehta is not properly installed and can't run.

While the installation procedre the command "./run.sh config" emits the error message:
BUILD FAILED
.../setup.xml:96: The following error occurred while executing this line:
.../install/db/db-specific/list-db-hsqldb.xml:3: .../install/client/db/hsqldb-intern not found.

Reason: the directory .../install/client/db/hsqldb-intern is missing.
Workaround: before running "./run.sh config" create the dierectory manually.

Required Fix: for a HSQL-based installation the directory must be created programmatically.
