In DeepaMehta "Hide" and "Delete" are generic commands. "Hide" 
removes a topic (or association) from a topic map. "Delete" removes 
the topic from the corporate memory (and thus from all topic maps). 
Search topics (tons) are different: they provide a "Remove" command 
with the semntic of "Delete".

Feature Request: The "Remove" command of search topics should 
have a context-sensitive semantic: if the search topic is part of the 
published version of the topic map "Hide" should be performed, 
"Delete" otherwise.

Background: Without this feature the publisher of a demo topic map is 
not able to place persistent search topics on the demo map: If a demo 
user removes a search topic it is vanished for all subsequent demo 
users. The proposed feature request arises from this demo mode 
scenario but might be generally useful.
