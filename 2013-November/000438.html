I like Malte's suggestion to render the association's value (as entered in the detail panel), along with the association. This approach is straight-forward and builds on what is there already. We could revitalize old #34 (Webclient: Enable "labelling" of an Association resp. "Roles") for this task, and I would realize it soon. Technically I would render the assoc value direct-on-canvas (not as DOM) as this is the more easy approach this time.
