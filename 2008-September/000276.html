Summary: Recipient List rendering not up-to-date

Details: The rendering of a Recipient List must be updated in these situations:

- An email address is assigned to a person or an institution (resp. an assigment is removed or retyped)

- A person or institution is created or deleted (or retyped)

Implementation note: possibly it is more reasonable not to react on all of these situations but to update the rendering on-demand, that is every time a recipient list is clicked. This would require a slight framework extension: the introduction of a "topicSelected" hook.
