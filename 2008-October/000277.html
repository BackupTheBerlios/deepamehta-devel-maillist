Summary: Several bugs in feature #4299 (handling of result sets)

Details: There are several bugs regarding the new search tons, in particular the result lists displayed in the property panel:

a) The result list is not always sorted. It should be sorted alphabetically.

b) The topic icons do not always appear in the result list (broken image). This is the case for tons obtained by a "What's related?" command.

c) The topic icons should also be clickable and reveal the respective topic (as the hyperlinks do). This is particularily an issue for topics with no name (which are not selectable for the moment).

d) The rendering of the result list is not always up-to-date. E.g. when double-clicking the ton (which triggers the search again) the result list is only up-to-date when the user selects the ton again (including deselecting it before).

e) Usability: if the result of a "What's related?" command is a single ton, it should be selected automatically in order to let the user see the result instantly. (Note: there are cases, however, when the result set comprises several tons. This might be the case when navigating "by Association Type".)

f) Eventually, the tons "Show result" command must be disposed because it is obsolete.
