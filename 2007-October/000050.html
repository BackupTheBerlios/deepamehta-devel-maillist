Summary: Some associations lack privilegs for non-administrators

Details: There is an error in the access control system: at least in 2 situations programatically created associations lack privilegs for non-administrators:

1) the "Type Access" association which is programtically created between a user and a type once the user created a new type manually.

2) the "Derivation" association which is programtically created between a type and its supertype once the user created a new type manually.

As a result the user is not able e.g. to delete these programtically created associations, but he should. This error applies to users which are not administrators.

For detailed info, follow this link:
http://developer.berlios.de/bugs/?func=detailbug&bug_id=12290&group_id=5174
