Hi Jürgen,

sorry for breaking the Box Renderer installation!

The Box Renderer plugin is now split into 2 separate projects:
	https://github.com/jri/dm4-box-renderer-canvas.git
	https://github.com/jri/dm4-box-renderer-dom-git

Both realize Torsten's new Look&Feel, one via Canvas rendering, the other via DOM based rendering.
The corresponding Maven artifacts are
	de.deepamehta/deepamehta-box-renderer-canvas
	de.deepamehta/deepamehta-box-renderer-dom

The former "deepamehta-boxrenderer" and "deepamehta-box-renderer" artifacts are obsolete and can be removed.

I updated the Jenkins jobs accordingly. There are 2 Box Renderer jobs now.
As Post Steps I entered (but not sure if this is correct):
	/usr/local/bin/ci_copy_plugin.sh dm4-box-renderer-canvas
	/usr/local/bin/ci_copy_plugin.sh dm4-box-renderer-dom

Both plugins could but should not be installed at the same time. For our nightly build demo server I would recommend to deploy the dm4-box-renderer-dom project as this is the new one with cool DOM rendering. Testing this is appreciated.

Many Thanks + Cheers,
Jörg
