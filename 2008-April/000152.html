Summary: Calendar app.: view not updated once person is renamed

Details: The calendars week-view is not updated immediately if the person-name of an appointment-attendee has been changed. This is particularily an issue for newly created persons (via appointment: "Assign Attendee" -> "Create Person") because in the moment of creation the person has no name yet.

This bug is not serious. The user can easily force a refresh of the calendar view by e.g. toggling to month-view and back to week-view. But it _is_ a bug and should be fixed.
