Summary: Calendar app.: view not up-to-date when location changes

Details: The calendars week-view is not updated immediately if the location-name of an appointment-location has been changed. This is particularily an issue for newly created locations (via appointment: "Assign Location" -> "Create Location") because in the moment of creation the location has no name yet.

Also, calendars week-view is not updated immediately if another location is assigned to an appointment (via appointment: "Assign Location").

These bugs are not serious. The user can easily force a refresh of the calendar view by e.g. toggling to month-view and back to week-view. But these _are_ bugs and should be fixed.
